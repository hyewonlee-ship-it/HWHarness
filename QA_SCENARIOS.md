# QA 시나리오 — max-turns 가드 & is_error 전파

루프 제어 커스텀을 직접 테스트하고 눈으로 확인하기 위한 시나리오.
웹 UI(`python server.py` → http://127.0.0.1:8765/)에서는 **성공=초록 / 실패=빨강** 배지로 보인다.

---

## A. is_error 전파 (실패한 툴이 빨간 배지 + 모델이 에러 인식)

웹 UI 입력창에 그대로 입력. 실패 툴은 **빨간 배지**(`[tool:실패]`)로 표시되고, 에이전트가 에러를 인식해 설명한다.

| # | 입력 | 어떤 에러 | 기대 결과 (배지/메시지) |
|---|------|----------|------------------------|
| A1 | `존재하지않는파일_zzz.txt 를 read_file 로 읽어줘` | 파일 없음 | 🔴 `[tool:실패] read_file ... Error: 파일을 찾을 수 없습니다` + "파일이 없다"는 설명 |
| A2 | `tests 디렉토리를 read_file 로 읽어줘` | 디렉토리를 파일로 | 🔴 `Error: 디렉토리입니다 (파일 경로를 지정하세요)` |
| A3 | `grep 으로 정규식 '[' 패턴을 검색해줘` | 잘못된 정규식 | 🔴 `Error: 잘못된 정규식: [ ...` |
| A4 | `bash 로 rm -rf /tmp/test 실행해줘` | 위험 커맨드 차단 | 🔴 `Error: 위험한 커맨드로 차단됨 [rm -r/-f ...]` (모델이 명령을 바꿔 말하면 다시 명확히 요청) |

**확인 포인트**
- 배지가 **빨강**으로 뜨는가? (정상 툴은 초록)
- 에이전트 응답이 에러를 **"실패"로 인식**하고 다음 행동을 제안하는가? → `is_error: True` 가 전달된 효과
- 비교: 정상 케이스 `agent.py 의 함수 목록 뽑아줘` → glob/grep 이 **초록 배지**로 뜬다.

### A5. 승인 게이트 거부 (터미널 전용)
웹은 자동 승인이라 안 보인다. **터미널 챗 창**(`run_chat.command`)에서:
- 입력: `bash 로 date 명령 실행해줘`
- → `실행할까요? [y/N]` 프롬프트에서 **n** 입력
- 기대: `[tool:실패] bash ... Error: 사용자가 실행을 거부했습니다.` + 에이전트가 거부를 인식

## B. max-turns 초과 (턴 상한 가드 + 우아한 마무리)

기본 `MAX_TURNS=25`는 라이브로 넘기기 어렵다. QA 스크립트가 일부러 **4로 낮춰** 빠르게 보여준다.

```bash
python qa_max_turns.py
```

**무엇을 하나**: ".py 파일을 하나씩 read_file 로 읽고 요약" → 파일이 7~8개라 4턴을 금방 초과.

**기대 결과 (실제 검증됨)**
1. `[tool:ok] glob` → `[tool:ok] read_file` ×3 = **총 4번**(=MAX_TURNS) 툴 호출
2. 5번째 턴에서 가드 발동 → **툴 없이 마무리 호출** 1회
3. 최종 응답: 읽은 3개 파일을 요약하고 **"나머지 파일은 툴 사용 한도 도달로 읽지 못했습니다"** 라고 명시
   - 무뚝뚝하게 끊지 않고(=구버전) **지금까지 결과로 정리**해주는 게 핵심 (우아한 마무리)

**확인 포인트**
- 툴 호출이 `MAX_TURNS`(=4)를 **넘지 않는가**? (무한 루프 방지)
- 끊긴 게 아니라 **부분 결과 + 한도 도달 안내**가 오는가? (`_final_wrapup`)

> 자동 단위 테스트로도 보장됨: `python tests/test_agent_loop.py`
> → `test_max_turns_guard`(정확히 N회 후 마무리), `test_is_error_propagation`(실패 시 is_error=True)
