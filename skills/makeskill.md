# 파일이름 추천 가이드

## 목적
사용자가 파일을 생성하거나 저장할 때 적절한 파일이름을 추천한다.

## 추천 원칙

### 1. 명확성 (Clarity)
- 파일의 **목적과 내용**이 이름에서 바로 드러나야 한다
- 예: `report.txt` ❌ → `2026_sales_report.txt` ✅

### 2. 간결성 (Conciseness)
- 필요 이상으로 길지 않게 (30자 이내 권장)
- 예: `quarterly_financial_analysis_report_for_2026.txt` ❌ → `2026_q1_financial_report.txt` ✅

### 3. 일관성 (Consistency)
- **파일 종류별 네이밍 컨벤션** 준수:
  - 데이터/결과물: `[날짜]_[주제].[확장자]` 예: `2026-06-08_analysis.csv`
  - 코드/스크립트: `[기능]_[용도].[확장자]` 예: `parse_json.py`
  - 문서: `[제목]_[버전].[확장자]` 예: `guide_v1.0.md`

### 4. 확장자 올바르게
- 파일 포맷에 맞는 확장자 사용 (`.txt`, `.csv`, `.json`, `.md`, `.py` 등)

### 5. 특수문자 회피
- 띄어쓰기: 언더스코어(`_`) 또는 하이픈(`-`) 사용
- 스페셜 문자(`*`, `?`, `/`, `\`, `:`, `"`, `|`, `<`, `>`) 절대 금지

## 상황별 추천 예시

### 데이터 분석/리포트
- 주제: "분기별 판매 현황"
- 추천: `2026_q2_sales_summary.csv` 또는 `sales_report_2026-06.xlsx`

### 설정/구성 파일
- 추천: `config.json`, `settings.ini`, `.env.example`

### 백업/아카이브
- 추천: `backup_2026-06-08.tar.gz`, `archive_v2.1.zip`

### 임시 파일
- 추천: `temp_test.txt`, `draft_memo.md` (접두사 `temp_` 또는 `draft_` 사용)

### 로그 파일
- 추천: `app.log`, `error_2026-06-08.log`, `debug.log`

## 체크리스트
파일이름 제안 전에 다음을 확인하세요:

- [ ] 파일의 **목적**이 명확한가?
- [ ] **날짜**가 필요한가? (필요하면 `YYYY-MM-DD` 또는 `YYYY_MM_DD` 형식)
- [ ] **버전**이 필요한가? (필요하면 `_v1.0` 또는 `_v2`)
- [ ] 파일 **확장자**가 맞는가?
- [ ] **특수문자**나 **띄어쓰기**는 없는가?
- [ ] **길이**가 적절한가? (30자 이내)

## 추천 요청 시 제공하면 좋은 정보
1. 파일의 **목적/내용** 설명
2. **파일 포맷** (텍스트, CSV, JSON, 바이너리 등)
3. 필요시 **생성 날짜/버전** 정보
4. **기존 파일 구조**나 네이밍 패턴 (일관성 유지)
